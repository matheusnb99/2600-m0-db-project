OPÉRATION BLACKVAULT — Microservices TAJ

Un microservice par rôle PostgreSQL · la sécurité est portée par la base, pas par l'application
Se connecter (service d'auth · port 3000) ↗

Principe

Chaque service ci-dessous est la même application, déployée avec une chaîne de connexion PostgreSQL différente (un rôle par service). Aucune règle de sécurité n'est codée dans le site : c'est le SGBD qui décide. Un service compromis ne peut donc jamais dépasser les droits de son rôle.

Accès lié au rôle : après connexion, un agent est automatiquement dirigé vers le microservice de son rôle. S'il tente d'ouvrir le service d'un autre rôle, il obtient une page « accès interdit » — et la base bloquerait de toute façon la requête.

① RBAC — accès aux tables

Le rôle de connexion (GRANT) détermine quelles pages répondent. Sans droit sur la table, la base renvoie « accès refusé » 🔒.

② Bell-LaPadula / RLS — filtrage des lignes

L'habilitation de l'agent connecté (NC<CD<SD<TSD) filtre quelles lignes sont visibles dans les tables autorisées (No Read Up).

③ Vues anonymisées

Les rôles de contrôle (auditeur, CNIL) n'accèdent qu'à des données pseudonymisées et à des statistiques agrégées — jamais la donnée brute.

Service d'authentification — port 3000 (point d'entrée unique)

Il vérifie les identifiants, émet un jeton (cookie d'hôte valable sur les 7 services), puis redirige automatiquement l'agent vers le microservice de son rôle (port 3000 + role_id → opj : 3002, magistrat : 3003…). Ouvrir un service sans être connecté redirige vers cette page de login ; tenter d'ouvrir le service d'un autre rôle affiche « accès interdit ».

Comptes de démonstration

Un compte par rôle (mot de passe commun Blackvault2026!). Se connecter avec l'un d'eux dirige vers le service de son rôle.

Matrice d'accès — rôle × page

R lecture  ·  W écriture  ·  RW lecture + écriture  ·  🔒 bloqué par la base (RBAC / Bell-LaPadula)
Les rôles de contrôle (auditeur, CNIL) lisent des vues anonymisées.

Démo : se connecter une fois via le service d'auth (port 3000) avec nicole.laurent@interieur.gouv.fr / Blackvault2026!, puis ouvrir les services et comparer ce qui s'affiche de l'un à l'autre — la session (cookie) est partagée. Le bandeau en haut de chaque service indique le rôle PostgreSQL de connexion.
